백엔드 학습시간을 가져보자!
Node.js 기반 백엔드 학습 여정과 새로운 기술 도전에 대한 경험을 공유합니다.
#도전
2024-11-26

요즘 리액트 네이티브를 학습하면서 개발하면서 자연스럽게 백엔드 API를 연결하는 일이 잦아졌습니다. 그러다 보니 백엔드 개발을 다시 학습해야겠다는 필요성을 느끼게 되었고, 오랜만에 관련 학습을 시작하게 되었습니다. 오랜만에 글을 쓰는 것 같아 조금 설레기도 합니다.

과거에는 Node.js의 대표적인 웹 프레임워크인 Express를 사용해 서버를 개발하고, SQL을 작성해 데이터베이스를 구축했었습니다. 그 당시에는 빠르고 간단하게 개발하는 것이 목표였지만, 요즘은 기술 트렌드가 많이 변했음을 실감하고 있습니다. 특히, 최근의 대부분의 기업이 TypeScript(TS)를 적극적으로 사용하고 있어, 이제는 TypeScript 없이 개발을 논하기 어렵다는 생각이 들었습니다.

TypeScript를 선택한 이유는 명확합니다. 정적 타입을 제공함으로써 코드의 안정성을 높이고, 코드 작성 시점에서 오류를 방지할 수 있다는 점은 큰 장점입니다. 또한 대규모 프로젝트에서 협업할 때, 명확한 데이터 구조를 정의할 수 있어 코드 가독성과 유지보수성이 크게 향상됩니다. 이러한 이유로 TypeScript 기반의 백엔드 프레임워크인 NestJS를 학습하기로 결정했습니다.

NestJS는 모듈화된 구조와 강력한 의존성 주입(Dependency Injection) 시스템을 제공하기 때문에, 대규모 애플리케이션에서도 유지보수가 용이한 코드베이스를 작성할 수 있다는 점이 매력적이었습니다. 학습을 시작하면서 느낀 것은 NestJS가 백엔드 개발에 있어 생산성을 크게 향상시키는 도구라는 점이었습니다.

백엔드 학습을 시작하면서 살펴보니 MSA 환경, Redis, 인증(Authentication), 보안(Security), 소켓 프로그래밍(WebSocket), 그리고 도커(Docker), AWS 등등 공부해야 할 것들이 정말 많다는 생각이 들었습니다. 다소 부담스러울 수 있는 목록이지만, 새로운 기술을 익혀가는 과정은 늘 흥미롭고 도전적입니다.

다행히 예전에 학습했던 백엔드 경험 덕분에 NestJS 공식 문서를 읽거나, 오류를 해결하는 데 있어 큰 어려움은 없는 것 같습니다. 오랜만에 백엔드 환경에서 코드를 작성하며 느끼는 것은, "한 번 익힌 기술은 몸에 남아 있다"는 것입니다. 물론 새로운 기술을 익히고, 적응하는 데에는 시간이 걸리겠지만, 그 과정 자체가 개발자로서 성장하는 시간이 아닐까 생각합니다.

앞으로 백엔드 학습도 꾸준히 진행하며, 경험했던 시행착오와 해결 방법, 그리고 NestJS를 활용한 다양한 기술 구현 방법들을 블로그에 정리해볼 생각입니다. 이번 학습이 제가 작성하는 API의 품질을 한층 더 끌어올리는 데 기여하길 기대하며, 이 글을 마칩니다.
