Git 푸시 중 "RPC failed; HTTP 400" 오류 해결하기
three.js와 GLB 파일을 활용하여 웹사이트를 개발하던 중, Git을 사용해 프로젝트 변경 사항을 원격 저장소에 푸시하려 할 때 "RPC failed; HTTP 400" 오류가 발생했습니다. 이 문제를 해결한 방법을 공유합니다.
#트러블 슈팅
2024-09-02

문제 상황

최근 three.js를 활용하여 웹사이트를 개발하면서, 3D 모델링 파일인 GLB 파일을 프로젝트에 추가하였습니다. 작업을 마친 후, 프로젝트 변경 사항을 Git을 통해 원격 저장소에 푸시하는 과정에서 다음과 같은 오류 메시지가 발생했습니다.

error: RPC failed; HTTP 400 curl 22 The requested URL returned error: 400
send-pack: unexpected disconnect while reading sideband packet
fatal: the remote end hung up unexpectedly

이 오류는 대개 대용량 파일을 푸시할 때 발생하며, 네트워크 문제나 Git 설정에 문제가 있을 때도 나타난다고한다.

원인 분석

  1. 대용량 파일

    : GLB 파일은 일반적으로 크기가 클 수 있으며, 이러한 대용량 파일을 Git을 통해 푸시할 때 HTTP 요청이 실패할 수 있습니다.

  2. Git 설정 부족

    : Git의 기본 설정으로는 큰 파일을 전송하는 데 제한이 있을 수 있습니다. 특히, http.postBuffer 설정이 낮은 경우 대용량 파일 전송 시 문제가 발생할 수 있습니다.

해결 방법

  1. Git 설정 조정: Git의 http.postBuffer 설정을 조정하여 Git이 더 큰 패킷을 전송할 수 있도록 합니다. 기본값은 1MB로 설정되어 있어 대용량 파일을 푸시할 때 문제가 발생할 수 있습니다.

  2. 설정 조정 명령어:

    git config --global http.postBuffer 524288000
    위 명령어는 http.postBuffer 값을 500MB로 설정하여 Git이 더 큰 패킷을 처리할 수 있도록 합니다.

  3. 다시 푸시 시도: 설정을 조정한 후, 다시 푸시 명령어를 실행하여 문제를 해결했는지 확인합니다.

    git push origin main

이 설정 조정 후, 원격 저장소에 성공적으로 푸시할 수 있었고, 문제를 해결할 수 있었습니다. 이 방법은 대용량 파일을 포함한 푸시를 할 때 유용하게 사용될 수 있습니다.

결론

RPC failed; HTTP 400 오류는 Git의 기본 설정으로 인해 발생할 수 있으며, http.postBuffer 값을 조정하여 해결할 수 있습니다. 대용량 파일 푸시 시 이 방법을 활용하면 문제를 해결할 수 있으므로, 비슷한 문제가 발생할 때 이 방법을 시도해 보세요!!
