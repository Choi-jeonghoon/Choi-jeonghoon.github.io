Utterances는 Github Issue를 통한 댓글 작성 기능

지금 개발이되어있는 이 블로그 또한 해당 어플리케이션을 활용해서 기능을 구현해놨다. 선택한 이유는 정적인 사이트에서 많이 사용한다고 했기도 하고 감정표현 같은것도 가능하기 때문에 선택하게 되었다. 또한 가장 만족스러운것은 git Issue를 활용하는 거기 때문에 git에 알림설정을 해두면 나의 메일로 확인이 가능하다.

사용방법은 아래와같다.

  • github Utterances 애플리케이션을 설치 해준다.

  • 레포지토리 권한 부여는 사용하고자하는 레포지토리만 하거나 아니면 전체로 하거나 둘중하나 선택한다.

여기부터는 코드를 작업해준다.

export default function GitComment() {
const ref = useRef<HTMLDivElement>(null)
// 다크모드를 추가했기때문에 해당 상태관리를 나는 추가한것이다. 없어도 무방하다.
const { theme } = useThemeStore()

useEffect(() => {
   if (ref.current === null) return
   
   const utterances: HTMLScriptElement = document.createElement('script')
   const attributes = {
      src: 'https://utteranc.es/client.js',
      repo: 'test/test.github.io',
      'issue-term': 'pathname',
      label: 'Comment',
      theme: theme === 'dark' ? 'github-dark' : 'github-light',
      crossorigin: 'anonymous',
      async: 'true',
}

   Object.entries(attributes).forEach(([key, value]) =>
      utterances.setAttribute(key, value),
    )

  ref.current.appendChild(utterances)
  
   return () => {
      if (ref.current?.hasChildNodes())
        ref.current.removeChild(ref.current.childNodes[0])
    }
}, [theme]) 
  return <div ref={ref} />
}

위와같이만 하나의 compoent를 작성해두면 해당 기능을 사용하는 곳에서 불러서 사용하면된다.

주의사항

  • 이슈 관리

    • 각 페이지마다 이슈가 생성되기때문에 ,GitHun 저장소의 이슈가 많아 질수있다.(관리를 잘해줘야된다.)

  • 만약 사이트가 엄격한 CSP가 설정되어 있다면, 외부 스크립트를 로드할 수 있도록 해당 정책에 예외를 추가해야된다고한다!

  • SEO

    • 해당 기능은 클라이언트 사이드에서 동적으로 로드되는것이 때문에, 검색 엔진 크롤러가 댓글 내용을 인덱싱을 못하는 수도 있다고한다.

  • 비동기 로딩

    • 해당 스크립트가 비동기로 로드되기 때문에 네티워크 상태나 GitHub의 상태에 따라 스크립트 로딩에 시간이 걸릴 수 있다. 이러한 경우에는 댓글 영역이 늦게 나타날수도 있다.

  • 테마 변경 관련

    • 위 코드를 보면 테마변경이 되고있는데 그것이 변경될 때마다 useEffect가 실행된다. 이러하면 페이지가 렌더링된 후에 테마가 변경되면서 스크립트가 도시 로드된다. 이로 인해 댓글이 사라졌다가 나타나는 현상이 발생 할 수도있다.

장점

혼자 작성하고 아는 지인들에게 작성을 부탁하면서 사용해봤을때 이슈 현황을 바로바로 확인할수있다는것이 가장 좋게 다가왔다.

느낀점

아직 많은 사람들이 보는것도아니고 많은 글이 존재하지 않기때문에 이슈관리 부분에서의 얼마나 문제가 생길지 모르겠지만 추후에 문제가 생긴다면 데이터베이스를 생성하여 댓글 기능을 구현하는 방식으로 바꿔보는 생각을 하게되었다.
